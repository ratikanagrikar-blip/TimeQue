⏰ EventPulse – Product Requirements Document

Version: v1.0 | Date: April 2026 | Status: Draft

1. Product Overview

What the Product Does

EventPulse is a lightweight Progressive Web App that allows users to create events and receive notifications at 24h, 6h, and 30m before the event. Works offline with no backend.

Target Users

  • Students tracking deadlines
  • Professionals managing meetings
  • Privacy-focused users
  • Users on low-end devices

Core Value

  • Proactive notifications
  • Offline-first
  • No login required
  • Fast & lightweight

2. Goals

  • Create event in under 10 seconds
  • Notifications at 24h, 6h, 30m
  • Offline functionality
  • Installable PWA
  • Local storage via IndexedDB

3. Core Features

Event Management

  • Create, edit, delete events
  • Sort by date
  • View upcoming/past events

Notifications

ReminderTime
Reminder 124 hours before
Reminder 26 hours before
Reminder 330 minutes before

UI/UX

  • Dark/Light mode
  • Mobile-first design
  • 3 screens: Home, Add Event, Settings

Offline Support

  • Works without internet
  • Uses service worker caching

4. Functional Requirements

  • Validate event title and date
  • Store events in IndexedDB
  • Schedule notifications on save
  • Cancel notifications on delete
  • Prevent duplicate notifications

5. Non-Functional Requirements

  • Load time under 1.5s
  • Bundle size under 150KB
  • Accessible UI (WCAG compliant)
  • Works on Chrome, Edge, Safari

6. Technical Architecture

  • Frontend: HTML, CSS, JavaScript
  • Storage: IndexedDB
  • PWA: Service Worker + Manifest

7. User Flow

  1. User opens app
  2. Adds event
  3. Grants notification permission
  4. Receives reminders

8. Future Enhancements

  • Cloud sync
  • Recurring events
  • Calendar view
  • Event sharing
